Important Tutorials

The following are some of the commands that might be needed for data analysis on the cluster.

Convert data from DICOM to NIFTI

On the cluster dcm2niix is used for conversion of raw DICOM data into Nifti. In order to use the functionality, the conda virtual environment needs to be activated on the cluster with the command source /usr/local/apps/psycapps/config/conda_bash_update dcm2niix. Locate where the raw data from the scanner has been saved for the particular scans. The raw data is under /MRIRaw/MRIRaw[PI_workspace_number]. Please confirm with Ari about the exact location of the raw data. Create an output folder in the corresponding workspace /MRIWork/MRIWork[PI_workspace_number]/[user_folder]. For example For example /MRIWork/MRIWork10/pv/[user_folder] is a sample workspace where the output folder needs to be created. Then the following command will be issued to perform the Dicom to Nifti Conversion.
dcm2niix -z y -f %p_%t_%s -o [output_folder] [dicom_directory_in_MRIRaw]

Use SPM and FSL on the cluster

For using SPM and FSL GUI on the cluster, you need to access the cluster through VNC viewer. Please refer to the Cluster Access section for the details of how to use GUI for accessing the cluster. After loggin on through VNC issue the command source /usr/local/apps/psycapps/config/matlab_bash_update 2019a in the terminal. This would select the required version of Matlab on the cluster. With the command matlab in the terminal the Matlab window will open. The corresponding SPM packages can be downloaded from this link.

FSL 6 is also installed on the cluster. The FSL GUI can be operated by the command fsl from the terminal.

Running Heudiconv BIDS conversion

Heudiconv or Heuristics-based Dicom Conversion pipeline is implemented on the cluster which converts raw DICOM data into BIDSformat, ready for open science data sharing platforms. The codebase on the cluster can be found here. For users who want to try it out themselves there is a very handy tutorial with an example here. For other users please contact Ayan for setting up a specific pipeline for the required conversion.

Running job scheduler

The cluster is a combination of High Performance Computational nodes meant for big data analysis and parallel processing. Ideally all R, SPM (Matlab) and FSL analysis should be run parallely on the worker nodes of the cluster which makes the performance much faster. The maintainance of the job queue on the worker nodes is done by a job scheduler software Sun Grid Engine. All users are encouraged to run all SPM or FSL analysis scripts in this fashion with the help of BASH wrapper scripts and submit scripts. All BASH scripts that is needed for using the job scheduler are available here, here and here. Please contact Ayan for setting up specific scripts for the required parallelization.
